## 产品定位（已定）

- 陪伴优先，其次工具；长期记忆要可控、可编辑、可追溯。
- 隐私第一：默认本地；云端只做显式授权后的增强。
- 禁止静默上云；需要「问询确认」再出站。

## 运行形态（已定）

- 首平台：Windows。
- 办公核显：可用，但接受体验上限；用小模型 + 强量化 + 短上下文保流畅。
- 混合推理：简单/陪伴 → 本地；复杂 → 弹窗问询 → 同意后再云端；返回结果建议本地人设润色。

## 模式开关（建议写进 README）

- 仅本地（默认或强隐私用户首选）
- 询问后云端（推荐默认商业形态）
- 始终询问（更严格，凡出站都确认）
- 可选极客档：自动路由上云（必须非默认 + 强警告）

## 问询弹窗必含信息

- 将上传什么（默认：仅当前条；附加上下文要可选且说明）
- 不上传什么（按实际架构写死，避免夸大）
- 目的（云端强推理 / 自托管算力）
- 本次 / 本会话 / 全局（全局要二次确认）

## 技术栈方向（已定）

- GGUF 本地推理（Windows：NVIDIA CUDA 体验档优先写清）
- 本地向量记忆 + SQLite（或等价），数据目录如 `%LOCALAPPDATA%\...`
- 语音 ASR/TTS：可 Phase 2，避免拖垮 MVP
- 记忆：摘要 / 归档 / 过期 / 去重，不能无脑堆

## 必须并入的「硬项」（已定）

- 安全与责任：危机内容分级与固定边界话术（不上云也能做）
- 评测与回归：小对话集 + 人格/记忆/安全用例
- 数据寿命：导出/导入、加密与密钥备份说明、schema 迁移

## MVP 交付顺序（建议）

1. 本地对话 + 会话落盘 + 人格 YAML + 角色锁
2. 记忆写入（可开关）+ 检索注入 + 用户删改
3. 导出/导入包
4. 混合云端：问询 + 最小上传 + 本地润色 + 出站可见/可关
5. 语音与延迟优化
6. 商业化预留：人设包版本、可选自托管服务端

## 仓库命名备选（专业）

- `private-companion-engine`
- `offline-companion-core`（当前）
- `on-device-companion-ai`

## 换仓库后建议立刻做的 5 件事

1. 新建仓库 + `.gitignore`（排除模型权重、venv、本地 DB）
2. README：Privacy Model（默认本地、问询出站、无静默上传）
3. 定 MVP 验收：核显档「能聊」+ 记忆「重启仍在」+ 云端「必确认」
4. 定数据目录与导出格式（单文件包 + manifest）
5. 建 10–30 条回归对话（人设、记忆、安全、混合问询）
